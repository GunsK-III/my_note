# Python 项目目录结构
* 在开始 Python 项目之前，IDE 会创建一个项目文件夹`pythonProject1`，
我们的项目文件都包含在这个文件夹中。  
* 根据不同的场景，创建不同的项目文件并进行配置是很好的习惯。（不推荐使用默认项目名称）

        pythonProject1/       <-- 您的项目根目录
        ├── .venv/            <-- Python虚拟环境（隔离项目依赖）
        ├── src               <-- src 表示源码（Source Code）
        │   ├── main.py       <-- 项目的入口文件
        │   ├── utils.py      <-- 工具类
        │   └── requirements.txt  <-- 项目依赖的包列表
        └── .idea/            <-- PyCharm等IDE的配置文件（通常是隐藏的） 

1. 重点是`.venv`文件。  
>* `.venv` 文件夹是 Python虚拟环境（Virtual Environment） 的核心所在。 
>  该文件夹是一个独立的、隔离的“小房间”，专门为 `pythonProject1` 项目服务。
>* 最重要的，整个项目中所依赖的第三方包都安装在`lib`的`site-packages`下。  
>  如果在没有创建或激活虚拟环境时，使用<code>pip install</code> 安装，会把包安装在系统默认路径下。
   往往是`C:\Users\AppData\Local\Programs\Python\Python311\Lib\site-packages\`  
   清华源镜像安装方式：<code>pip install -i https://pypi.tuna.tsinghua.edu.cn/simple <包名> </code>
>* 进入虚拟环境，在CMD中执行：<code>.venv\Scripts\activate</code>，在pycharm的终端窗口默认激活虚拟环境。  
>  退出虚拟环境，在CMD中执行：<code>deactivate</code>

        pythonProject1/
        ├── venv/                       # 虚拟环境根目录
        │   ├── Lib/                    # Windows: Python库目录
        │   │   └── site-packages/      # ★ 第三方包实际安装在这里 ★
        │   ├── Scripts/                # Windows: 可执行文件目录
        │   │   ├── activate            # 激活脚本
        │   │   ├── pip.exe             # 虚拟环境专用的pip
        │   │   └── python.exe          # 虚拟环境专用的Python
 
2. `.idea` 文件是PyCharm IDE 的配置文件。不用管它。
3. `src` 中放入项目代码（推荐）。


# PyCharm 编辑项目
1. 编辑器右下角可以选择不同环境的解释器。也就是说可以尝试在一个项目中，使用另一个项目的配置执行代码。
2. 在 PyCharm 的终端中，输入的命令可以直接在虚拟环境中执行，比如使用 <code>pip</code> 命令安装第三方库。
>* 在系统的 CMD 中，需要先在`.venv`目录下，用`.\activate`命令对虚拟环境进行激活，激活成功后，命令行前会出现(myenv)。如果不激活，则默认使用的是计算机的全局环境。在虚拟环境中完成任务后还需要使用`deactivate`命令退出虚拟环境。
